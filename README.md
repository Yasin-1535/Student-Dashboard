# Student CGPA Management SaaS

A modern, full-stack Student CGPA Management SaaS application built from scratch.

## Tech Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, React Router DOM, Axios, Chart.js
- **Backend**: Python, Flask, Flask REST API, SQLAlchemy ORM, SQLite, Flask-JWT-Extended, Flask-CORS

## 🏗 System Architecture

This SaaS application is built using a modern decoupled client-server architecture. The backend acts as a stateless RESTful API, while the frontend functions as a rich Single Page Application (SPA).

### Architectural Breakdown

```
        +----------------------------------------+
        |             User Browser               |
        +----------------------------------------+
                            │  ▲
                            ▼  │  HTTPS / JSON
        +----------------------------------------+
        |       Frontend Layer (React + TS)      |
        |  Tailwind CSS (UI) | Axios (API)       |
        +----------------------------------------+
                            │  ▲
                            ▼  │  REST API Requests
        +----------------------------------------+
        |        Backend Layer (Flask API)       |
        |  Routes | Blueprints | JWT Validation  |
        +----------------------------------------+
                            │  ▲
                            ▼  │  SQL Queries (Python)
        +----------------------------------------+
        |            SQLAlchemy ORM              |
        +----------------------------------------+
                            │  ▲
                            ▼  │  SQLite Driver
        +----------------------------------------+
        |          SQLite Database File          |
        +----------------------------------------+
```

1. **Frontend Layer (React & TypeScript)**: Serves the static assets and renders the responsive Single Page Application. It uses Vite for building, Tailwind CSS for visual style, and Axios to coordinate API requests.
2. **Backend Layer (Flask REST API)**: Hosts modular Flask Blueprints representing domain routing contexts. It implements JWT authentication checks and coordinates request decoding, error handling, and business operations.
3. **Database Layer (SQLAlchemy ORM & SQLite)**: Maps relational tables to Python objects dynamically using SQLAlchemy. The lightweight SQLite engine acts as the persistent datastore.
4. **Authentication Layer**: Coordinates secure role validation via JWT tokens. During mock execution, an automated decorator bypasses strict authentication to support frictionless admin and student dashboard simulation.
5. **API Layer**: Exposes standardized JSON REST endpoints (`/api/students`, `/api/subjects`, `/api/marks`, `/api/analytics`) to serialize database states cleanly over HTTP.

## 🔄 Detailed Workflows

### Authentication & Token Validation Flow
```
User Login
  │
  ▼
API Request to backend (/auth/login)
  │
  ▼
JWT Access Token Generated by Flask
  │
  ▼
Token Returned in Response & Saved in LocalStorage
  │
  ▼
Subsequent API Requests include "Authorization: Bearer <token>"
  │
  ▼
Backend interceptor validates Token Signature
  │
  ▼
Database query completes matching request details
  │
  ▼
Stateless response returned as JSON
  │
  ▼
State hooks update in React and UI re-renders
```

### ⚙️ How the Application Works

#### 1. Student Creation & Enrollment Flow
* **Initiation**: The Admin clicks the **Add Student** action, opening the dialog containing profile details (name, enrollment number, department, batch, and semester) along with an optional marks draft grid.
* **Payload Structure**: Staged fields are aggregated into a single nested payload containing student metadata alongside an array of initial `marks` records.
* **Backend Processing**: The `/api/auth/register` controller receives the request. It creates a secondary security `User` record, flushes the transaction to generate the `Student` ID, loops through the provided marks, and inserts both student and subject-marks concurrently inside a single transaction before committing.

#### 2. Marks Submission Flow
* **Data Capture**: In **Manage Marks**, the user selects a Student and Subject, and enters Internal (out of 50) and External (out of 50) scores.
* **Calculation & Grades**: Upon post execution, the backend calculates `total_marks = internal + external`. This is routed to `calculate_grade_points()` which resolves the Grade and Grade Point (e.g. `total >= 90` yields `Grade: O, Point: 10`).
* **Upsert**: Checks for existing entries for the given student/subject combination. If found, it updates the record; otherwise, a new `Marks` record is saved and committed.

#### 3. GPA & CGPA Calculation Flow
* **Math Formulas**:
  $$\text{GPA} = \frac{\sum (\text{GradePoint} \times \text{Credits})}{\sum \text{Credits}}$$
  
  $$\text{CGPA} = \frac{\sum \text{GPA}}{\text{Total Semesters}}$$
* **Engine**: The server executes `calculate_cgpa` inside `backend/app/utils/cgpa_calc.py`. It pulls the complete list of academic marks for the target student, resolves the subject credits, and dynamically returns the correct weighted grade average.

#### 4. Dashboard Analytics Flow
* **Data Aggregation**: When opening the dashboard, `/api/analytics/dashboard` aggregates data points using SQLAlchemy queries (`func.count`, `func.sum`).
* **Processing**: Generates calculated scores like Total Students, Average CGPA, Pass Rates (percentage of students with no `F` grade), and sorts a Top 5 students list.
* **Presentation**: The frontend translates this aggregated payload into dynamic canvas structures using Chart.js on the Analytics screen.

## How to Run Locally in VS Code

### Backend Setup
1. Open a terminal and navigate to the `backend` folder:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment (optional but recommended):
   ```bash
   python -m venv venv
   # On Windows:
   venv\Scripts\activate
   # On Mac/Linux:
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Run the Flask application:
   ```bash
   python run.py
   ```
   *The backend will run on `http://localhost:5000`.*

### Frontend Setup
1. Open a new terminal and navigate to the `frontend` folder:
   ```bash
   cd frontend
   ```
2. Install Node dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *The frontend will run on `http://localhost:5173`.*

## Deployment Configuration
- Ensure to change `sqlite:///` to a production database like PostgreSQL when deploying to Render or Railway.
- Use `gunicorn` for the Flask production server (`gunicorn run:app`).
- For the frontend, build the production bundle using `npm run build` and deploy the `dist` folder to Vercel or Netlify.

## Roles
- **Admin**: Can view all students, subjects, marks, and view analytics. (Must set role to 'admin' manually in DB or via a super admin endpoint).
- **Student**: Default role upon registration. Can view their own profile, current CGPA, and marks.
